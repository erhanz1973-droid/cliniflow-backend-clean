Connecting Facebook Messenger

Page admin requirements

  • Use a Facebook profile that is Admin (or has full control) on your clinic Page.
  • Business Manager access alone is not enough unless that profile can manage the Page in Meta settings.
  • If you manage multiple clinics, connect the Page that matches this Clinifly clinic.

Messenger must be enabled

  • Your Facebook Page should have Messenger turned on (default for most Pages).
  • In Meta Page settings, check Inbox / Messaging is available for the Page.

Retry permissions

Click Retry permissions if you previously denied access or need to grant Page permissions again. Meta will show the consent screen a second time — accept all requested Page and messaging permissions.

“No Pages found” — usually not a hard error

If Facebook returns no Pages to select, it usually means one of the following — not a permanent Clinifly block:
  • You signed in with the wrong Facebook account (personal profile that does not manage the clinic Page).
  • Your account is missing Page admin role on the clinic Page.
  • Permissions were denied or partially accepted during OAuth — use Retry permissions.
  • The Page was never created — create one at facebook.com/pages/create, then connect again.

Troubleshooting

  • Connect Facebook — start or add another Page connection.
  • Retry permissions — re-authorize after changing roles or denying scopes.
  • Messenger diagnostics — checks whether the stored Page token can reach Meta (after a Page is connected).
  • Reset state — clears stuck OAuth UI on this browser only; does not remove a connected Page from the server.

Patient messages already appear in Clinifly only after a Page is connected and webhooks are subscribed successfully.
